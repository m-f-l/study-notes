    类型断言

    mfl 2022/9/10

    # 概念&语法

    • 概念：用来手动指定一个值的类型，在某些情况下，我们会比 TS 更清楚一个数据的类型。这种时候，我们就可以使用断言来告诉 TS：相信我，我知道自己在干什么。

    • 语法：

      1. 值 as 类型
2. <类型>值

    # 使用场景

    # 1. 将一个联合类型断言为其中一个类型

    当ts不确定一个联合类型的变量到底是哪一个类型的时候，我们只能访问此联合类型的所有类型中共有的属性或者方法。

    function getLength(something: string | number): number {
    const result = something.length
    return result
}
// Property 'length' does not exist on type 'number'.

    使用断言：

    function getLength(something: string | number): number {
    const result = (something as string).length
    return result
}

    注意：在 tsx 语法 中只能用as type（尖括号语法与 jsx 语法冲突）

    例如：

    let name: string | number;
let myName: number = (<string>name).length;
// JSX element 'string' has no corresponding closing tag.'</' expected.

    # 2. 将一个父类断言为更加具体的子类

    详见参考 (opens new window)

      class ApiError extends Error {
    code: number = 0
  }

  class HttpError extends Error {
    statusCode: number = 200
  }

  function isApiError(error: Error) {
    if(typeof (error as ApiError).code ==='number') {
      return true;
    }
    return false;
  }

  // 父类Error中可能没有code属性，直接error.code会报错，需要通过断言获取code 即（error as ApiError).code

    # 3. 将任何一个类型断言为any

    我们在使用js进行开发时，有时候会在window对象上添加新的属性和方法，这个属性在全局就能访问了，但是在ts中会报错，这个时候我们就需要使用断言any了

    window.foo = 1;
// Property 'foo' does not exist on type 'Window & typeof globalThis'.

    使用断言：👇

    (window as any).foo = 1

    # 4. 将any断言为一个具体的类型

    看一下这段代码，mySum求和函数，返回的是number,但是ts指定返回的是类型，所以不会报错，但是这样子就会导致很多功能在编译阶段暴露出的错误没法暴露出来。这个时候需要使用断言将any断言成具体的类型，恢复它在编译阶段报错的功能。

    function mySum(a: number, b: number): any {
  return a + b
}
const sum = mySum(9, 8)
console.log(sum.length)

    解决方法：👇

    function mySum(a: number, b: number): any {
  return a + b
}
const sum = mySum(9, 8) as number
console.log(sum.length)

//Property 'length' does not exist on type 'number'.

    # 5. 类型断言的限制

    是不是任何一个类型都可以被断言为任意一种类型呢？不是

    • 如果A兼容B或者B兼容A,那么A能够被断言为B,B也能被断言为A (这里的兼容简单来说就是：A兼容B就是指类型A是类型B的子集)
    interface Animal {
  name: string;
}

interface Cat {
  name: string;
  run(): void;
}

function testAnimal(animal: Animal) {
  return (animal as Cat)
}

function testCat(cat: Cat) {
  return (cat as Animal)
}

    animal as Cat 因为父类可以被断言为子类

    interface Animal {
  name: string;
}
interface Cat extends Animal {
  run(): void
}

    cat as Animal 子类被断言为父类（子类拥有父类的属性和方法）

    • 如果类型A不兼容B并且B不兼容A,那么A不能被断言为B,B 也不能被断言为A.
    function test(num: number) {
  return (num as string)
}
// error

    # 6. 双重断言

    • 任何类型都可以被断言为any,any可以被断言为任何类型，使用双重断言把任何一个类型断言成任何另一个类型
    function mytest(num: number) {
  return (num as any as string)  // 被断言为string类型
}

    # 7. 类型断言不会进行类型转化

    • 类型断言只在TS编译时有效果，在编译结果中会被删除，不会影响编译结果的类型
    function mytest(num: number) {
  return (num as any as string)
}
const num: number = 1
const str  = mytest(num)
console.log(str) // 实际上还是number类型

    所以，需要转换类型

    function mytest(num: number) {
  return String((num as any as string))
}
const num: number = 1
const str  = mytest(num)
console.log(str)

    # 8. 类型断言vs类型声明

    实际上，类型声明的使用比类型断言要更严格

    • A断言为B: 需要满足A兼容B或者B兼容 A
    • A赋值给B: 需要满足B兼容A
    interface Animal {
    name: string;
}
interface Cat {
    name: string;
    run(): void;
}

const animal: Animal = {
    name: 'tom'
};
let tom = animal as Cat;

    Animal兼容Cat ,所以可以将animal断言为Cat赋值给tom.

    interface Animal {
      name: string;
  }
interface Cat {
    name: string;
    run(): void;
}

const animal: Animal = {
    name: 'tom'
};
let tom: Cat = animal;
//Property 'run' is missing in type 'Animal' but required in type 'Cat'

    如果直接声明tom为Cat类型，会报错，因为Cat不兼容Animal
